## Component Composition

One of MDX's powerful features is composing multiple components together:

### Example: Interactive Card

```mdx
import Button from '../../components/shared/Button.astro';
import Badge from '../../components/shared/Badge.astro';

<div style="border: 1px solid var(--border); padding: 1.5rem; border-radius: 0.5rem;">
  <h3>Featured Content</h3>
  <Badge text="New" variant="accent" />
  <p>This is an interactive card with multiple components.</p>
  <Button href="/blog" variant="primary">Learn More</Button>
</div>
```

## Advanced JSX Patterns

### Conditional Rendering

Use conditional rendering to show different content based on conditions:

```mdx
---
featured: true
---

{frontmatter.featured ? (
  <Badge text="Featured Post" variant="accent" />
) : (
  <Badge text="Regular Post" variant="default" />
)}
```

### Mapping Over Arrays

You can map over arrays to render lists of components:

```mdx
---
tags: ["mdx", "astro", "components"]
---

<div style="display: flex; gap: 0.5rem; flex-wrap: wrap;">
  {frontmatter.tags.map((tag) => (
    <TagPill tag={tag} href={`/blog?tag=${tag}`} />
  ))}
</div>
```

### Complex Calculations

Perform calculations and display results:

```mdx
---
wordCount: 2500
---

Estimated reading time: {Math.ceil(frontmatter.wordCount / 200)} minutes
```

## Dynamic Content Patterns

### Date Formatting

Format dates dynamically:

```mdx
Published: {new Date(frontmatter.date).toLocaleDateString('en-US', {
  year: 'numeric',
  month: 'long',
  day: 'numeric'
})}
```

### Conditional Components

Show components based on frontmatter:

```mdx
---
series: "MDX Mastery"
seriesPart: 2
---

{frontmatter.series && (
  <div style="background: var(--bg-secondary); padding: 1rem; border-radius: 0.5rem; margin: 1rem 0;">
    <p><strong>Series:</strong> {frontmatter.series} - Part {frontmatter.seriesPart}</p>
  </div>
)}
```

## Component Props and Styling

### Inline Styles

Use inline styles for quick styling:

```mdx
<div style="display: flex; gap: 1rem; margin: 2rem 0;">
  <Button href="/blog" variant="primary">Primary</Button>
  <Button href="/blog" variant="secondary">Secondary</Button>
</div>
```

### CSS Classes

If your components support class props, you can pass them:

```mdx
<Badge text="Custom" variant="accent" class="my-custom-class" />
```

## Advanced Markdown + Component Mixing

### Components in Lists

You can use components within markdown lists:

```mdx
## Features

1. Standard markdown works
2. <Button href="/blog" variant="primary">Interactive buttons</Button>
3. Components in lists
```

### Components in Callouts

Embed components within markdown directives:

:::tip
Try using <Button href="/blog" variant="secondary">components</Button> inside callouts!
:::

## Component Best Practices

1. **Import at the top** - Always import components right after frontmatter
2. **Use semantic HTML** - Wrap components in appropriate HTML elements
3. **Keep it accessible** - Ensure components are keyboard navigable
4. **Test thoroughly** - Test components in both dev and build modes

## Performance Considerations

- Components are rendered at build time (for static sites)
- Large component trees may impact build performance
- Consider lazy loading for heavy interactive components
